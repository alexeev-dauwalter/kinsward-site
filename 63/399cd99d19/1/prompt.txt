# Task: Set up kinsward-site repository

## Context

We're creating a public website + release hosting repo for the game Kinsward (a Dwarf Fortress-inspired colony sim). The game source lives in a private repo. This public repo serves two purposes:
1. Documentation site built with Astro Starlight, deployed to GitHub Pages
2. GitHub Releases hosting for game binaries (cross-repo published from the private repo's CI)

**Owner**: alexeev-dauwalter
**Repo**: kinsward-site (public, already initialized on GitHub)
**Domain**: kinsward.dwltr.life
**Private game repo**: the current repo you're working in

## Step 7: GitHub Actions deploy workflow

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy Site

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
      - run: npm ci
      - run: npm run build
      - run: echo "kinsward.dwltr.life" > dist/CNAME
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## Step 8: Cross-repo release job (in the GAME repo)

In the **current private game repo**, modify `.github/workflows/release.yml` to add a job after `build` and `release`:

```yaml
  publish-to-site:
    needs: [build, release]
    runs-on: ubuntu-latest
    steps:
      - name: Download all build artifacts
        uses: actions/download-artifact@v4
        with:
          path: artifacts
          merge-multiple: true

      - name: Rename artifacts for public release
        run: |
          cd artifacts
          for f in my_game-*; do
            newname=$(echo "$f" | sed 's/my_game/kinsward/')
            mv "$f" "$newname" 2>/dev/null || true
          done
          ls -la

      - name: Create release in kinsward-site
        uses: softprops/action-gh-release@v2
        with:
          repository: alexeev-dauwalter/kinsward-site
          token: ${{ secrets.SITE_REPO_TOKEN }}
          tag_name: ${{ github.ref_name }}
          name: "Kinsward ${{ github.ref_name }}"
          body: |
            ## Kinsward ${{ github.ref_name }}

            ### Downloads
            - **Windows** (x86_64): `kinsward-${{ github.ref_name }}-windows-x86_64.zip`
            - **Linux** (x86_64): `kinsward-${{ github.ref_name }}-linux-x86_64.tar.gz`
          files: |
            artifacts/kinsward-*.tar.gz
            artifacts/kinsward-*.zip
```
